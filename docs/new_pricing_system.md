# Новая система ценообразования тарифов

## Обзор

Система тарифов была переработана для поддержки динамического ценообразования на основе количества устройств. Теперь администратор задает базовую цену за 1 устройство и скидку на дополнительные устройства, а итоговая цена рассчитывается автоматически.

## Изменения в структуре данных

### Модель Tariffs

Добавлены новые поля:
- `base_price` (переименовано из `price`) - базовая цена за 1 устройство
- `device_discount` - скидка на дополнительные устройства в процентах (0-100)

### Метод расчета цены

```python
def calculate_price(self, device_count: int = 1) -> int:
    """
    Рассчитывает цену тарифа для указанного количества устройств.
    Формула: базовая_цена + (доп_устройства * базовая_цена * (1 - скидка/100))
    """
    if device_count <= 0:
        return 0
    
    if device_count == 1:
        return self.base_price
    
    additional_devices = device_count - 1
    discount_multiplier = (100 - self.device_discount) / 100
    additional_cost = additional_devices * self.base_price * discount_multiplier
    
    return int(self.base_price + additional_cost)
```

### Примеры расчета

**Пример 1:**
- Базовая цена: 100₽
- Скидка: 25%
- Количество устройств: 3

Расчет:
- 1 устройство: 100₽
- 2 дополнительных устройства: 2 × 100₽ × (1 - 25/100) = 2 × 100₽ × 0.75 = 150₽
- **Итого: 100₽ + 150₽ = 250₽**

**Пример 2:**
- Базовая цена: 200₽
- Скидка: 50%
- Количество устройств: 2

Расчет:
- 1 устройство: 200₽
- 1 дополнительное устройство: 1 × 200₽ × (1 - 50/100) = 1 × 200₽ × 0.5 = 100₽
- **Итого: 200₽ + 100₽ = 300₽**

## API изменения

### Новые endpoints

1. `GET /pay/tariffs/calculate?device_count={count}` - получение тарифов с рассчитанными ценами
2. `GET /pay/{tariff_id}?email={email}&device_count={count}` - покупка тарифа с указанием количества устройств

### Обратная совместимость

- Старый endpoint `/pay/tariffs` продолжает работать
- Свойство `price` в модели возвращает `base_price` для совместимости
- При отсутствии `device_count` используется значение по умолчанию = 1

## Изменения в интерфейсе

### Новый компонент TariffSelector

- Ползунок для выбора количества устройств (1-10)
- Сетка тарифов в 2 столбца
- Квадратные карточки тарифов
- Отображение скидки на дополнительные устройства
- Выделение популярного тарифа

### Особенности UI

- Автоматическое обновление цен при изменении количества устройств
- Отображение цены за месяц для многомесячных тарифов
- Инициализация количества устройств из текущей подписки пользователя

## Миграция данных

### SQL скрипт

```sql
-- Добавляем новое поле device_discount
ALTER TABLE tariffs ADD COLUMN device_discount INTEGER NOT NULL DEFAULT 0;

-- Переименовываем price в base_price
ALTER TABLE tariffs RENAME COLUMN price TO base_price;
```

### Настройка скидок

Рекомендуемые скидки для существующих тарифов:

```sql
UPDATE tariffs SET device_discount = 25 WHERE name = 'Месячный';     -- 25% скидка
UPDATE tariffs SET device_discount = 20 WHERE name = 'Квартальный';  -- 20% скидка  
UPDATE tariffs SET device_discount = 15 WHERE name = 'Полугодовой';  -- 15% скидка
UPDATE tariffs SET device_discount = 10 WHERE name = 'Годовой';      -- 10% скидка
```

## Администрирование

### FastAdmin интерфейс

Обновлены поля для отображения:
- `base_price` вместо `price`
- `device_discount` для настройки скидок

### Рекомендации

1. Устанавливайте базовую цену как стоимость для 1 устройства
2. Скидка должна быть в диапазоне 0-100%
3. Больше скидка = больше экономия для пользователей с несколькими устройствами
4. Рекомендуется тестировать различные скидки для оптимизации конверсии

## Технические детали

### Совместимость

- Старые активные тарифы продолжают работать
- При автопродлении используется сохраненный `hwid_limit`
- Новые покупки используют выбранное количество устройств

### Производительность

- Расчет цены происходит на лету
- Кэширование тарифов на фронтенде
- Минимальное количество API запросов

### Безопасность

- Валидация `device_count` на бэкенде
- Ограничение максимального количества устройств
- Проверка корректности расчетов
