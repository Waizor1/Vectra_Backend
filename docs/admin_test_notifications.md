# Админская команда тестирования уведомлений

## Обзор

Команда `/test_notification` позволяет администраторам тестировать любые типы уведомлений в системе TVPN, отправляя их конкретным пользователям.

## Использование

### Запуск команды
```
/test_notification
```

### Пошаговый процесс

1. **Выбор типа уведомления**
   - 🔄 Пробный период
   - 💰 Подписка  
   - 📱 Общие

2. **Выбор конкретного уведомления**
   Зависит от выбранного типа (см. ниже)

3. **Ввод ID пользователя**
   - Числовой ID: `123456789`
   - Username: `@username` или `username`

4. **Ввод дополнительных параметров** (если требуются)
   Формат зависит от типа уведомления

5. **Подтверждение и отправка**

## Типы уведомлений

### 🔄 Пробный период
- **Продление пробного периода** - требует: `days` (число дней)
- **Предоставление пробного периода** - без параметров
- **Скоро истекает пробный период** - без параметров  
- **Окончание пробного периода** - без параметров
- **Нет пробного периода** - требует: `hours_passed` (часов прошло)

### 💰 Подписка
- **Истекает подписка** - без параметров
- **Автоматический платеж** - без параметров
- **Успешное продление с баланса** - требует: `days amount` (дни и сумма)
- **Ошибка автопродления** - без параметров
- **Ключ отключен** - без параметров

### 📱 Общие
- **Активация ключа** - без параметров
- **Реферальный платеж** - требует: `referral_id amount` (ID реферала и сумма)
- **Регистрация по рефералу** - требует: `referral_id` (ID реферала)
- **Напоминание о реферале** - требует: `days` (число дней)

## Примеры использования

### Тест продления пробного периода
1. `/test_notification`
2. Выбрать "🔄 Пробный период"
3. Выбрать "Продление пробного периода"
4. Ввести ID пользователя: `7843161268`
5. Ввести количество дней: `7`
6. Подтвердить отправку

### Тест реферального платежа
1. `/test_notification`
2. Выбрать "📱 Общие"  
3. Выбрать "Реферальный платеж"
4. Ввести ID получателя: `123456789`
5. Ввести ID реферала и сумму: `987654321 500`
6. Подтвердить отправку

## Отмена операции

В любой момент можно:
- Использовать кнопку "❌ Отменить"
- Написать `/cancel`

## Логирование

Все тестовые уведомления логируются с префиксом `bot_admin_test_notifications` для отслеживания в логах.

## Требования

- Права администратора в системе
- Существующий пользователь для отправки уведомления
- Корректные параметры для уведомлений, требующих дополнительных данных

## Обработка ошибок

Система показывает подробные сообщения об ошибках:
- Пользователь не найден
- Неверный формат параметров  
- Ошибки отправки уведомлений
- Проблемы с Telegram API
