## Гайд по админке Directus (TVPN)

### 0) Режим гайда для команды (шаги + скриншоты)
1. **Навигация** — слева основные разделы и папки. Ориентир: папка «Основное».
   Скриншот: `assets/guide-navigation.png`
2. **Главный дашборд** — ключевые метрики и графики.
   Скриншот: `assets/guide-dashboard.png`
3. **Список пользователей** — фильтры, поиск, статусы, быстрый доступ к карточке.
   Скриншот: `assets/guide-users-list.png`
4. **Карточка пользователя** — вкладки и подсказки, где менять лимиты/подписку.
   Скриншот: `assets/guide-user-edit.png`
5. **Промо и призы** — см. разделы Promo Codes / Prize Wheel для типовых операций.

### 1) Куда заходить и что смотреть
- **Users** — основная карточка пользователя: подписка, лимиты, баланс, блокировки.
- **Active Tariffs** — текущие активные тарифы (LTE‑лимиты, мультипликаторы, usage).
- **Tariffs** — справочник тарифов и цен.
- **Promo Codes / Promo Batches / Promo Usages** — промокоды, их партии и история использования.
- **Prize Wheel Config / Prize Wheel History** — настройки «колеса» и история выпадений.
- **In-App уведомления** — всплывающие popup-уведомления в Mini App (раздел «Основное»).
- **Connections** — данные для графиков подключений.
- **Processed Payments** — обработанные платежи.

### 2) Частые действия (как работать)
**Поменять LTE лимит пользователю**
1. Открыть Users → найти пользователя.
2. Изменить `lte_gb_total`.
3. Сохранить — Directus вызовет backend для синхронизации.

**Продлить/ограничить подписку**
1. Users → `expired_at` или `hwid_limit`.
2. Сохранить — backend синхронизирует RemnaWave.

**Обновить LTE лимит активного тарифа**
1. Active Tariffs → `lte_gb_total`.
2. Сохранить — backend синхронизирует LTE.

**Поменять LTE лимит на триале**
1. Открыть служебную singleton-настройку `tvpn_admin_settings`.
2. Изменить `trial_lte_limit_gb` в ГБ.
3. `1.0` = стандартный 1 ГБ на триале; `0` отключает LTE на триале.
4. Backend перечитывает настройку с коротким кешем, без деплоя.

**Промокод**
1. Promo Codes → создать запись.
2. В `code_hmac` можно указать «сырой» код — хук преобразует в HMAC.

**Колесо призов**
1. Prize Wheel Config → изменить вероятность или приз.
2. Сумма вероятностей активных призов должна быть ≤ 1.0.
3. Для `subscription` нужно указывать целое число дней в `prize_value`.

**Всплывающее уведомление в Mini App (in-app popup)**
1. **In-App уведомления** (раздел «Основное») → **Создать запись**.
2. Заполнить: `Заголовок`, `Текст уведомления`.
3. Указать **расписание**: `Показывать с` / `Показывать до`.
4. Опционально: `Макс. показов на пользователя`, `Макс. показов на сессию`, `Авто-скрытие (сек)`.
5. `Активно = true` → сохранить.
6. Закладки: **«Уведомления: активные»** (зелёная) и **«Уведомления: неактивные»** (красная) для быстрой фильтрации.
7. История показов — коллекция `notification_views` (служебная, только чтение).

**Тарифы: как настраивать новый конструктор (рекомендуемый путь)**
1. Открыть модуль **Tariff Studio**: `/admin/tvpn-tariff-studio`.
2. Слева выбрать срок (`1 / 3 / 6 / 12 месяцев`). Отдельной family-card больше нет.
3. В редакторе менять продуктовые поля:
   - `Активен`;
   - `Порядок`;
   - `Срок, мес.`;
   - `Цена за 1 устройство`;
   - `Максимум устройств` (v1: 1–30);
   - `LTE включён`, `Цена LTE за 1 ГБ`, `Мин/макс/шаг LTE`;
   - `Бейдж` и `Подсказка` для витрины.
   - LTE по умолчанию должен оставаться включённым для всех сроков; базовый восстановленный прайс — `1.5 ₽/ГБ`, максимум конструктора — `500 ГБ`, шаг — `1 ГБ`.
4. Справа смотреть preview строк `1 / 2 / 5 / 10 / 30 устройств`.
   - `1` устройство = персональная подписка (`base`);
   - `2+` устройства = семейная ёмкость (`family`);
   - preview подтягивает backend `/admin/integration/tariffs/quote-preview` через Directus endpoint `/tariff-studio/quote-preview`, поэтому warnings/blocking errors совпадают с backend-валидацией.
5. Нажать **Сохранить**. Directus-hook `remnawave-sync` вызывает backend `/admin/integration/tariffs/compute-pricing`, сохраняет derived `base_price`/`progressive_multiplier`/legacy anchors и блокирует явно невалидные значения.
6. Raw collection **Tariffs** остаётся advanced/fallback для Manager/Admin. Служебные поля `progressive_multiplier`, `final_price_default`, `final_price_family`, `family_plan_enabled` не являются основным UX; ориентир — Tariff Studio.

### 3) Дашборды и метрики
Раздел **Аналитика** (`/admin/insights`) — список дашбордов и панелей.
API для графиков: `/admin-widgets/*`
- `total-users`
- `active-users`
- `inactive-users`
- `registered-users`
- `connections`

### 4) Роли (команда)
- **Administrator** — полный доступ.
- **Manager** — основной CRUD (users/active_tariffs/tariffs/promo/prize_wheel/**in_app_notifications**).
- **Viewer** — только чтение.

### 5) Полезные заметки
- Некоторые служебные таблицы помечены как «Служебное».
- Если что‑то не отображается, проверь права/политику.
 - Пороги алертов на “Главной” хранятся в коллекции `tvpn_admin_settings` (singleton, скрытая).

### 6) Быстрое восстановление (если админка “пустая”)

Симптом: после логина слева почти пусто (нет коллекций/папок), будто Directus “без разделов”.

Частые причины:
- у роли нет permissions на ваши коллекции (тогда Directus скрывает их из меню)
- у роли выключен `app_access`
- коллекции отмечены `hidden=true`, а доступ есть только к “служебным”

Решение (идемпотентно):
- настроить переменные `DIRECTUS_URL`, `DIRECTUS_ADMIN_EMAIL`, `DIRECTUS_ADMIN_PASSWORD`
- запустить `python scripts/directus_super_setup.py`
